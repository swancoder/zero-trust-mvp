==============================================================
CHAIN OF THOUGHT
==============================================================

R2DBC CHOICE (over blocking JDBC + Scheduler):
  WebFlux runs on Netty. Blocking a Netty I/O thread with JDBC causes thread
  starvation and degrades throughput. R2DBC is the only correct datasource for
  reactive runtime queries. Flyway continues to use JDBC (runs at startup, before
  any request is served).

CACHING (Mono.cache(Duration) over Spring @Cacheable / Caffeine):
  For a single cached value (all enabled policies), Reactor's Mono.cache(Duration)
  is the minimal, zero-dependency, idiomatic solution. Spring @Cacheable requires a
  CacheManager bean and external dependency. The cache is shared across all concurrent
  requests via the same Mono instance. TTL: 5 minutes (policy changes take effect
  within 5 minutes, no gateway restart needed).

FAIL-CLOSED (onErrorResume BEFORE cache):
  DB errors return empty policy list (deny all) and do NOT get cached.
  The error is handled before cache() wraps the Mono, so the empty list
  is cached (not the error). Next subscriber after TTL retries the DB.

FILTER ORDER (1):
  Spring Security's WebFilter runs before any Gateway GlobalFilter.
  SecurityContext is populated by the time ZteAuthorizationFilter runs.
  Order 1 = runs first in the gateway filter chain, before RequestAuditFilter
  (LOWEST_PRECEDENCE - 10) and before the routing filter.

ROLE EXTRACTION:
  Default Spring Security JwtGrantedAuthoritiesConverter looks for `scope` claim.
  Keycloak puts realm roles in `realm_access.roles`. Extracted directly via
  Jwt.getClaimAsMap("realm_access").get("roles") inside the filter.

PATH MATCHING:
  AntPathMatcher.match("/api/v1/service-a/**", "/api/v1/service-a/hello") = true.
  No path rewriting in the route — gateway path == service-a internal path.
  This keeps policies, routes, and service paths fully consistent.
