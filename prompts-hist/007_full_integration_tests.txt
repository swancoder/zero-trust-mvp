==============================================================
CHAIN OF THOUGHT
==============================================================

Container orchestration:
  Singleton pattern — all containers started in a static{} block in BaseZteIntegrationTest.
  All subclasses produce the same @DynamicPropertySource output → Spring test context cache
  creates ONE ApplicationContext for the entire integration run. Only ~15-30s startup cost
  for the Keycloak container, amortised across HappyPathIT + ZeroTrustBreachIT.

  WireMock runs in-process (no Docker) on a dynamic port. Gateway routes are overridden
  via @DynamicPropertySource to http://localhost:<wireMockPort>. This means service-a
  and service-b Docker images are NOT needed for the integration tests.

Why mocking R2DBC is wrong here (unlike unit tests):
  The policy engine uses PostgreSQL-specific R2DBC behaviour. H2 doesn't support the
  same dialect. Testcontainers PostgreSQL gives us a real DB, ensuring Flyway migrations
  run correctly and the policy cache (Mono.cache) is exercised with real SQL queries.

MtlsHttpClientConfig conditional:
  ReloadableSslContextFactory loads certs in its constructor. In tests, certs do not
  exist (gitignored). @ConditionalOnProperty(zte.mtls.enabled) allows disabling cert
  loading in the test profile while preserving production behaviour (matchIfMissing=true).
