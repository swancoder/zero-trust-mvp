  - service-a module: Spring Boot, GET /api/v1/service-a/hello, port 8081
  - Gateway route + docker-compose service-a entry

Critical fix (HTTP 500 → 403):
  Root cause: NettyRoutingFilter was still forwarding to service-a after the filter wrote a 403,
  because response.setComplete() does NOT suppress Gateway routing.
  Fix applied:
    1. Filter order: Ordered.HIGHEST_PRECEDENCE + 100 (runs before routing filters)
    2. exchange.getAttributes().put(GATEWAY_ALREADY_ROUTED_ATTR, true) in writeForbidden()
    3. response.writeWith(Mono.just(buffer)) with explicit JSON DataBuffer

Dual-datasource fix:
  Spring Boot 3.4 does not reliably auto-select JDBC DataSource for Flyway when
  spring-boot-starter-data-r2dbc is also on the classpath.
  Fix: explicit spring.flyway.url/user/password in application.yml
